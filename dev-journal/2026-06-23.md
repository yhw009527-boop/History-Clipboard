# 2026-06-23 开发日志

## 本次目标

- 按确认计划升级主窗口体验，并新增默认开启的常驻悬浮按钮。

## 完成事项

- 新增 `floatingButtonVisible` 设置，旧设置缺少该字段时默认开启。
- 新增独立悬浮窗页面，支持按钮形态和轻量历史面板形态。
- `Ctrl+Shift+V` 改为打开悬浮面板，托盘菜单新增主窗口、悬浮面板、显示/隐藏悬浮按钮入口。
- 悬浮面板支持查看最近记录，点击记录后写回系统剪贴板并收起。
- 主窗口增加悬浮按钮开关，并优化空状态、状态反馈、按钮可读性和小屏布局。
- 新增 M10 smoke 验收路径。
- 悬浮按钮支持拖动并保存位置，图标更换为用户提供的 `火焰帽下的自信表情.png`。

## 验证方式

- `npm.cmd run build`
- M3-M8 smoke 回归
- M10 floating smoke

## 验证结果

- `npm.cmd run build` 通过。
- M3-M8 smoke 回归通过。
- M10 smoke 通过，输出 `M10_SMOKE_TEST_FLOATING_WINDOW_OK`。
- 悬浮按钮拖动和自定义图标改动后，`npm.cmd run build` 与 M10 smoke 继续通过。
- 手动拖动实现后，M10 smoke 覆盖拖动位置变化，M3/M8 关键回归继续通过。
- 拖动卡顿反馈后，悬浮球拖动改为动画帧节流并仅在拖动结束保存位置；悬浮球改为透明圆形图标裁切。
- 继续处理悬浮球卡顿和点击面板后悬浮球卡进面板的问题：悬浮球与悬浮面板拆成两个独立页面，按钮窗口加载 `floating-button.html`，面板窗口加载 `floating.html`。
- 悬浮球拖动改为主进程按鼠标位置跟随，渲染进程只发送开始/结束拖动事件，不再暴露高频移动 IPC。
- M10 smoke 增加校验：悬浮球窗口必须包含按钮，悬浮面板窗口不得渲染悬浮球。
- 修复打包版拖动时报 `TypeError: Error processing argument at index 0`：拖动开始坐标和移动坐标增加有限数字校验，坐标无效时使用当前鼠标位置兜底或跳过 `setPosition`。
- 继续修复打包版拖动时报 `TypeError: Error processing argument at index 1`：悬浮窗默认位置、保存位置和拖动目标位置统一取整，`setPosition` 外层增加异常保护。
- “关于历史剪贴板”菜单改为自定义关于弹窗，显示版本号和“氧化物制作”。
- 已重新执行 `npm.cmd run package:win`，新包输出到 `release\历史剪贴板-win32-x64`。
- smoke 临时 userData 目录清理偶发 EPERM 警告，不影响各项验收结果。

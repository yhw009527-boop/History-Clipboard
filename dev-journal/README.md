# 开发日志规则

## 目的

`dev-journal` 用于记录每天的开发进展，保证项目按小步骤稳定推进，并保留每次决策和待办的依据。

## 文件命名

每天一个日志文件，命名格式：

```text
YYYY-MM-DD.md
```

示例：

```text
2026-06-20.md
```

## 记录要求

每次开发结束前，必须更新当天日志。日志至少包含：

- 本次目标
- 完成事项
- 验证方式
- 验证结果
- 遗留问题
- 下一步待办

## 工作边界

- 日志记录只反映已经发生的事实和已确认的计划。
- 未确认的想法必须标记为“待确认”。
- 不得把未经用户确认的功能写成已决定事项。

## 模板

新建当天日志时，使用 `dev-journal/_template.md` 作为模板。

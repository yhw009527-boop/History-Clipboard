# UI 设计规范

## 设计目标

界面应简洁、直观，适合快速查找和再次使用剪贴板历史。第一版不做复杂视觉效果，优先保证清晰、稳定、易用。

## 视觉风格

- 主色调使用淡蓝色。
- 背景保持清爽，避免过深或过重的颜色。
- 操作按钮应清晰可识别。
- 页面信息层级要明确。
- 卡片之间需要有足够间距，便于扫描。

## 主界面结构

- 顶部区域：
  - 搜索框。
  - 设置入口。
- 内容区域：
  - 历史记录卡片列表。
  - 默认按时间降序排列。
  - 置顶内容显示在普通内容之前。
- 卡片区域：
  - 文字内容显示预览。
  - 图片内容显示缩略图。
  - 提供置顶操作。
  - 提供删除操作。

## 交互规范

- 点击历史记录：将内容重新写入系统剪贴板。
- 点击置顶：切换该记录的置顶状态。
- 点击删除：删除该记录。
- 搜索输入：实时或触发式过滤文字历史记录，具体方式在实现阶段确认。

## 设置界面

设置至少包含：

- 保留时长：`1 天`、`3 天`、`5 天`。
- 开机自启状态。
- 快捷键说明：`Ctrl+Shift+V`。

## 暂不设计

- 复杂主题系统。
- 多颜色主题切换。
- 图片 OCR 搜索界面。
- 账号登录界面。
- 云同步界面。
